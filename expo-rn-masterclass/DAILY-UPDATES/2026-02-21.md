## 🧠 Pattern of the Day — Optimistic Updates with TanStack Query

When the user performs an action (like, bookmark, delete), update the UI immediately and reconcile with the server in the background.

```tsx
import { useMutation, useQueryClient } from "@tanstack/react-query";

export function useLikePost() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (postId: string) =>
      fetch(`/api/posts/${postId}/like`, { method: "POST" }),
    onMutate: async (postId) => {
      // Cancel outgoing fetches
      await queryClient.cancelQueries({ queryKey: ["posts"] });

      // Snapshot previous value
      const previousPosts = queryClient.getQueryData(["posts"]);

      // Optimistically update
      queryClient.setQueryData(["posts"], (old: any[]) =>
        old.map((post) =>
          post.id === postId ? { ...post, liked: true, likes: post.likes + 1 } : post
        )
      );

      return { previousPosts };
    },
    onError: (_err, _postId, context) => {
      // Rollback on error
      queryClient.setQueryData(["posts"], context?.previousPosts);
    },
    onSettled: () => {
      // Refetch to sync with server
      queryClient.invalidateQueries({ queryKey: ["posts"] });
    },
  });
}
```

**Why it matters:** Users perceive instant response. The app feels 10x faster.

## ⚠️ Deprecated Today

- **`TouchableOpacity`** — has been soft-deprecated for over a year. Use `Pressable` with `style={({ pressed }) => [...]}` for all new components. `Pressable` provides `onPressIn`, `onPressOut`, `android_ripple`, and proper accessibility.
